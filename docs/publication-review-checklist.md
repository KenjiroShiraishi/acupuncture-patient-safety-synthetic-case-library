# Publication Review Checklist

This checklist should be used before publishing any synthetic educational case.

## 1. Synthetic case status

- [ ] The case is clearly labeled as a synthetic educational case.
- [ ] The case is not presented as a real patient case.
- [ ] The case is not presented as a de-identified case report.
- [ ] The case is not presented as an adverse event database entry.

## 2. Privacy and re-identification risk

- [ ] No names are included.
- [ ] No exact age is included.
- [ ] No exact date is included.
- [ ] No location is included.
- [ ] No facility-specific information is included.
- [ ] No practitioner-identifying information is included.
- [ ] No occupation, family structure, or rare personal details are included.
- [ ] No direct quotations from real conversations are included.
- [ ] No rare combination of details could allow re-identification.

## 3. Clinical and safety content

- [ ] The safety issue is clear.
- [ ] The decision point is clear.
- [ ] The safer response is clear.
- [ ] The case does not minimize risk.
- [ ] The case does not exaggerate risk.
- [ ] The case does not provide specific medical advice for an individual patient.
- [ ] The case encourages appropriate reassessment or medical consultation when needed.

## 4. Tone and blame avoidance

- [ ] The case does not blame the patient.
- [ ] The case does not blame an individual practitioner.
- [ ] The case does not criticize a specific school, style, or facility.
- [ ] The case is written for learning, not accusation.
- [ ] The language is educational and neutral.

## 5. AI use

- [ ] AI was used only after the learning point was abstracted.
- [ ] AI output was reviewed by a human.
- [ ] The final text was edited by a human.
- [ ] The case does not contain hallucinated clinical claims.
- [ ] The case does not cite fabricated sources.

## 6. Final decision

- [ ] Approved for publication
- [ ] Needs revision
- [ ] Do not publish
